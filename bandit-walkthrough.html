▼

Bandit 12 — Compression Hell
Reversing a hexdump, then peeling many layers of compression
FILESYSTEMCRYPTO
▼
The file is a hexdump of data compressed multiple times (gzip, bzip2, tar). Strategy: reverse the hexdump with xxd -r, then repeatedly run file to identify the format and decompress accordingly until you hit ASCII text.
bash
mkdir /tmp/work && cd /tmp/work
cp ~/data.txt .
xxd -r data.txt > output.bin   # hexdump → binary
file output.bin                # → gzip
mv output.bin output.gz && gzip -d output.gz
file output                    # → bzip2
bzip2 -d output
file output.out               # → gzip
mv output.out data4.gz && gzip -d data4.gz
file data4                    # → tar
mv data4 data4.tar && tar -xvf data4.tar
file data5.bin                # → tar
tar -xvf data5.bin
file data6.bin                # → bzip2
bzip2 -d data6.bin
file data6.bin.out            # → tar
tar -xvf data6.bin.out
file data8.bin                # → gzip
mv data8.bin data9.gz && gzip -d data9.gz
file data9                    # → ASCII text, done
cat data9

Bandit 26 — Vim Shell Escape
Exploiting the 'more' pager to escape into vim then a real shell
SHELL
▼
bandit26's shell is more. If your terminal is tall enough, more exits immediately. Resize it to ~5 lines so more pauses. Then press v to open vim, set a real shell, and spawn it. Classic CTF shell escape.
bash + vim
# 1. Resize terminal to ~5 lines tall
# 2. SSH in — 'more' will pause instead of exiting
ssh -i bandit26.sshkey bandit26@bandit.labs.overthewire.org -p 2220

# Inside 'more' — press v to open vim
v

# Inside vim:
:set shell=/bin/bash
:shell                           # spawns bash as bandit26

# Now in bash — run the setuid binary for bandit27
./bandit27-do cat /etc/bandit_pass/bandit27
